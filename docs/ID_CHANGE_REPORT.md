# Отчет об изменении идентификаторов тир-листов

## Суть изменений
В рамках миграции `20260505080714_add_uuid` тип первичного ключа (`id`) таблицы `tier_lists` был изменен с целочисленного (`Int`) на `UUID`.

## Причины
1. **Безопасность**: Числовые ID позволяют легко перебирать записи (ID enumeration). UUID делает невозможным угадывание ссылок на чужие или приватные списки.
2. **Масштабируемость**: UUID позволяют генерировать идентификаторы на стороне клиента или в распределенных системах без риска коллизий и необходимости обращения к центральному счетчику.

## Оценка критичности
Изменение **не является ошибкой**. Это запланированное улучшение архитектуры.

## Дальнейшие шаги
Для обеспечения удобства пользователей (красивые ссылки) будет внедрена система "Slugs" (человекочитаемые URL), которая позволит использовать в ссылках названия списков вместо UUID, сохраняя при этом безопасность последнего как основного ключа в БД.
## Текущий статус
Система слагов внедрена. Теперь при создании тир-листа генерируется уникальный слаг на основе заголовка. Редактор и превью поддерживают оба формата ссылок (UUID и слаг). Ссылки в интерфейсе автоматически переключены на использование слагов там, где это возможно.
