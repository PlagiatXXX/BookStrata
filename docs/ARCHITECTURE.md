# Архитектура BookStrata Pro

## 1. Обзор
**BookStrata Pro** — это профессиональное Full-stack веб-приложение для организации книжных библиотек и создания интерактивных рейтингов (tier lists). Проект нацелен на любителей чтения и коллекционеров, предоставляя им инструменты для визуализации своих предпочтений.

## 2. Стек технологий и требования

### Системные требования
- **Node.js**: v18.0.0 или выше.
- **PostgreSQL**: v14.0 или выше.
- **Пакетный менеджер**: npm или pnpm.

### Основной стек
- **Frontend**: React 19, TypeScript 5.9, Vite 7, TailwindCSS 4, TanStack Query 5.
- **Backend**: Fastify 5.7, Prisma ORM, Zod, JWT.
- **Инфраструктура**: Cloudinary (изображения), Google Books API (данные).

## 3. Модульная структура и логика

### Frontend (`src/`)
- **App & Router**: Настройка React Router 7 с поддержкой защищенных маршрутов (`ProtectedRoute`) и прав администратора (`AdminGuard`).
- **Hooks & State**:
  - `useTierList`: Реализует сложную логику перемещения элементов. Использует `optimistic updates` для мгновенного отклика интерфейса.
  - `useAuth`: Управляет жизненным циклом сессии, автоматически обновляя состояние пользователя при загрузке.
- **UI System**: Набор атомарных компонентов в `src/ui`, построенных на принципах доступности (A11y).

### Backend (`backend/src/`)
- **Fastify Ecosystem**: Использование плагинов для инкапсуляции логики (Auth, Logger, CORS).
- **Prisma Schema**: Мощная типизированная модель данных. Модели `TierList` и `Tier` имеют связь "один ко многим", а `BookPlacement` служит связующим звеном для гибкого позиционирования.
- **Services**: Вся тяжелая логика вынесена в сервисы (`.service.ts`), что делает контроллеры тонкими и легко тестируемыми.

## 4. Ошибки и подводные камни (Pitfalls)

### 1. Состояние Drag-and-Drop
- **Проблема**: При быстром перемещении элементов состояние может десинхронизироваться с сервером.
- **Решение**: Использование `useAutoSaveOptimized` с механизмом подавления дребезга (debounce) и строгим порядком операций.

### 2. Лимиты Google Books API
- **Проблема**: Бесплатный уровень Google Books API имеет жесткие квоты.
- **Решение**: Реализовано проксирование через бэкенд с потенциальной возможностью кэширования в Redis.

### 3. Производительность Prisma
- **Проблема**: Вложенные инсерты (`createMany`) не возвращают ID созданных записей.
- **Решение**: Использование транзакций и последовательных сохранений там, где важны идентификаторы для фронтенда.

### 4. Конфликты вложенных стилей Tailwind 4
- **Проблема**: При использовании сложных 3D-компонентов стили могут накладываться друг на друга.
- **Решение**: Использование изолированных CSS-модулей для специфичных сцен (`BookScene.css`).

## 5. Лучшие практики проекта
- **Fail-Fast**: Валидация всех входящих данных на бэкенде через Zod схемы.
- **Type Safety**: Общие типы в папке `shared/` гарантируют, что фронтенд и бэкенд всегда говорят на одном языке.
- **Contextual Logging**: Использование кастомного логгера для отслеживания ошибок в продакшене с учетом контекста вызова.
